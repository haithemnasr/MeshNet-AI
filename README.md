## Overview

MeshNet AI simulates a resilient mesh communication network across 10 Tunisian cities. When disasters (floods, earthquakes, infrastructure failures) knock out nodes, trained RL agents (DQN and PPO) autonomously re-route messages through surviving nodes — outperforming classic algorithms in both delivery rate and energy efficiency.

| Algorithm | Delivery Rate | Avg Energy | Avg Hops |
|---|---|---|---|
| Flooding | ~40% | ~19.4 | ~5.2 |
| Dijkstra | ~82% | ~0.38 | ~3.1 |
| Greedy | ~76% | ~0.31 | ~2.8 |
| AODV | ~84% | ~0.36 | ~3.0 |
| **DQN (ours)** | **~100%** | **~0.13** | **~2.0** |
| **PPO (ours)** | **~98%** | **~0.14** | **~2.1** |

## Features

- **6 routing algorithms** benchmarked: Flooding, Dijkstra, Greedy, AODV, DQN, PPO
- **3 disaster scenarios**: Nabeul coastal flood, Kasserine earthquake, Tunis infrastructure failure
- **28-feature RL state space** with per-neighbor encoding for fine-grained decisions
- **Interactive Dash dashboard** with animated message routing and live KPIs
- **Multi-seed robustness evaluation** across 5 random seeds with error bars
- **Modular `src/` package** ready to extend with new cities, scenarios, or agents

## Quick Start

### 1. Clone & install

```bash
git clone https://github.com/haithemnasr/meshnet-ai.git
cd meshnet-ai
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
pip install -e .          # install src/ as an editable package
```

### 2. Train RL agents

This trains both DQN and PPO for 500 000 timesteps each (~15–40 min depending on hardware):

```bash
python train_rl_agents.py
```

Weights are saved to `models/dqn/dqn_final.zip` and `models/ppo/ppo_final.zip`.

### 3. Benchmark all algorithms

```bash
python comparison_test.py
```

Runs 5 random seeds, prints a summary table, saves charts to `data/results/comparison/`, and writes `data/dashboard_metrics.json`.

### 4. Launch the dashboard

```bash
python dashboard.py
```

Open **http://127.0.0.1:8050** in your browser.

> **Note:** Train the models first (step 2). The dashboard works without them but AI routing buttons will show a warning.

## RL Environment Details

**State space — 28 features:**

| Group | Features |
|---|---|
| Global (8) | Battery, num_neighbors, hops_ratio, distance_to_dest, progress, at_dest, best_neighbor_dist, avg_battery |
| Per neighbor × 5 (20) | dist_to_dest, battery, connectivity, is_valid |

**Action space:** 5 discrete actions — choose which sorted neighbor to forward to.

**Reward shaping:**

| Event | Reward |
|---|---|
| Delivery | +50 + efficiency bonus (up to +10) |
| Drop / max hops | −20 |
| Progress toward dest | +2 |
| Moving away | −1 |
| No progress | −0.2 |
| Energy cost | −0.05 × energy |

## Disaster Scenarios

### 🌊 Nabeul Coastal Flood
Progressive inward flooding from the coast. Affects: **Nabeul, Monastir, Mahdia, Sousse**. 65 % node failure rate per city with a guaranteed 1-node survivor.

### 🏔 Kasserine Earthquake  
Sudden seismic event (M5.5) with aftershocks. Affects: **Kasserine, Kairouan**.

### ⚡ Tunis Infrastructure Failure  
Critical hub cascade collapse. Affects: **Tunis, Ariana** — the most densely connected nodes.
